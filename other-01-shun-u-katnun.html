Шунн у катнун — Встреча 1

Тип: Презентация МИС Renovatio ветеринарной клинике
Клиника: Шунн у катнун (Собака и кошка), Ереван, Армения
Дата: 01.07.2026
Контакт: Татьяна @Tanda_Tank
Участники: менеджер Renovatio + директор клиники, врач, медсестра, администратор
Длительность: ~60 минут

Контекст

Первый созвон-демонстрация МИС Renovatio команде ветеринарной клиники «Шунн у катнун» (арм. «Собака и кошка», Ереван). Клиника работает в двух направлениях:

  • Частные приёмы: домашние питомцы, обычный поток
  • Социальная работа с бездомными животными: 50–70 животных в день, конвейерные процедуры (чипирование, стерилизация, бирки на ухо)

По словам менеджера, это первая ветеринарная клиника в Армении, которую внедряют в систему Renovatio.

Что обсуждалось

1. Запись пациентов

  • Два способа записи: предварительная (по звонку, через слоты) и живая очередь
  • Статусы визита: записан → на приёме → завершён
  • Администратор видит «маяки»: визиты без документов, без счёта
  • Возможность перетаскивать карточки между статусами вручную
  • Подтверждение записи галочкой — владелец сообщил, что придёт
  • Рекламные каналы: можно привязать источник первичного визита к карточке владельца и к конкретному визиту

2. Расписание врачей

  • Гибкие варианты: ежедневно / чётные-нечётные / по дням недели / произвольно / два через два
  • Разные слоты у разных врачей (например, консультация 30 мин, процедурный кабинет 5 мин)
  • Опция «без пересечения визитов» — защита от двойной записи. При необходимости отключается
  • Вид расписания: по врачам, по кабинетам, по оборудованию
  • Неделя целиком видна одним экраном; при наведении на слот — детали визита

3. Резерв

  • Если точная дата ещё неизвестна — можно поставить в резерв с интервалом «через N времени»
  • Резерв висит у врача как напоминание пока не запишут
  • Срочный резерв — флаг для приоритетного назначения при появлении окна

4. Карточка владельца и питомца

  • Несколько владельцев на одно животное: основной + дополнительные с комментарием («может принимать решения» / «только забрать»)
  • Актуально для бездомных: одна организация = 40 животных, у части из них по два куратора
  • Карточка питомца: порода, клеймо, чип, окрас, дата чипирования, фото
  • Номер ушной бирки для бездомных — поля нет. Обсуждались два варианта: добавить отдельное поле или переименовать «клеймо». Менеджер взяла уточнить у Георгия
  • Файлы: можно прикрепить к карточке владельца или к конкретному животному (анализы, подписи, документы)
  • Настраиваемые обязательные поля при регистрации (по умолчанию: имя + телефон; остальное — по желанию клиники)

5. Медицинская документация

  • Шаблоны (протоколы): клиника создаёт свои, с произвольными полями
  • Врачебные фразы-заготовки внутри протокола — выбрать одним кликом, не писать заново
  • Гибкость печати: весь документ или только нужные блоки (например, только рекомендации пациенту)
  • История болезни хранится в системе и привязана к визитам, питомцу, владельцу
  • Ветклиника запросила базовые ветеринарные шаблоны — менеджер обещала сделать к следующей встрече

6. Оплата и касса

  • Счёт формируется автоматически из услуг, добавленных к визиту
  • Частичная оплата, раздельная (наличные + безналичные), долги — всё отображается
  • Клиника спросила про армянские платёжные методы (DRAM, Apple Pay) — уточнить у Георгия

7. Статистика и отчёты

  • Есть отчёты по владельцам: пол, возраст, рекламные каналы, финансовая история
  • Клиника хочет отчёты по питомцам: вид животного, порода, пол, возраст, диагноз, популярные услуги — сейчас этого нет, нужно разрабатывать
  • Детализация: «самая популярная услуга → кого ею лечат → демография этих животных» — аналитики такого уровня пока нет
  • Динамика веса питомца в системе формально есть, но клиника хочет видеть изменение в динамике на временной шкале
  • Выгрузка отчётов в Excel

8. Вакцинация

  • Журнал вакцинации: список питомцев с датами прививок
  • При создании вакцинации можно сразу поставить резерв «перезвонить через месяц»
  • Клиника хочет вносить прививки, сделанные в других клиниках — для отслеживания следующей даты. Возможно, нужно уточнить

9. Уведомления и рассылки

  • Система поддерживает напоминания: за час, за день до визита
  • Рассылка по SMS / мессенджерам — теоретически возможна, но в Армении ещё не тестировалась. Менеджер обещала уточнить

Ключевые инсайты

# Инсайт Контекст
1 🔴 Поле «номер ушной бирки» — блокер для работы с бездомными 50–70 животных в день, основная идентификация — бирка. Без поля нужен workaround
2 🔴 Клиника требует аналитику по питомцам, а не только по владельцам «Не интересно, кто хозяин — интересно, какие животные с какими диагнозами». Сейчас системы такой нет
3 🔴 Первая ветклиника в Армении — нет локальных референсов Армянские платёжные методы (DRAM, Apple Pay), рассылки по локальным мессенджерам — всё «теоретически возможно, но не тестировалось»
4 🟡 Несколько владельцев на одно животное — реальный сценарий Организация + физлицо-куратор; муж + жена. Клиника уже думала об этом до демо
5 🟡 Шаблонные врачебные фразы — сильный аргумент для врачей Врач активно реагировал на возможность не печатать текст заново каждый раз
6 🟡 Динамика показателей питомца во времени Вес, результаты анализов — клиника хочет видеть не просто текущие значения, а изменение. Сейчас это не реализовано явно
7 🟢 Система в целом произвела хорошее впечатление «В целом мне всё очень нравится» (директор). Клиника готова продолжать — планируется следующая встреча и чат с командой

  • Уточнить у Георгия: армянские платёжные методы (DRAM, Apple Pay)
  • Уточнить у Георгия: рассылки через мессенджеры в Армении
